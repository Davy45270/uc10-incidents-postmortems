# UC10 Incidents & Postmortems

## Expression de besoin fonctionnelle
- **Objectifs** : gérer incidents, réaliser RCA, suivre actions correctives.
- **Acteurs** : SRE, Ops, Dev, support.
- **Flux clés** : détection → qualification → coordination → résolution → postmortem → actions.
- **Exceptions** : incidents majeurs avec cellule de crise; communication externe.
- **Critères d’acceptation** : workflow incident, templates postmortem, suivi actions avec échéances.

## Analyse ROI
- **Gains** : réduction du MTTR, baisse de la récurrence, capitalisation.
- **Coûts** : outillage ITSM/monitoring, formation RCA.
- **KPI cibles** : MTTR, taux de récurrence, % actions clôturées à temps.
